# 规则与注意事项

以下规则用于保证存档可靠性与回滚一致性：

## 1. 存档稳定性优先

- 保存前建议等待存档稳定（文件停止变化）。
- 若存档仍在写入，优先选择稍后保存，避免记录不完整的状态。

## 2. 脏工作区的处理原则

- 当检测到脏工作区时，涉及分支/路线切换、回滚等操作会要求明确选择：
  - 保存当前变更
  - 丢弃当前变更
  - 取消操作
- 不做自动决策，避免误丢有效存档。

## 3. 回滚需要建立新路线

- 回滚操作应在目标提交上创建并切换到新路线，避免覆盖原有历史。
- 如果未能提供新路线名，回滚将被拒绝。

## 4. 手动分支决策

- 自动分支检测暂不启用。
- 由玩家自行判断是否创建新路线，避免误切换到不期望的历史。

## 5. 游戏运行态的限制

- 某些游戏在运行时会将存档保护在内存中，磁盘文件仅为延迟写入的副本。
- 在此类游戏中，回滚可能需要先退出或重启游戏才能生效。
